# Abstract

The Fast Fourier Transform (FFT) is a cornerstone algorithm in digital signal processing, with applications spanning telecommunications, audio processing, radar systems, and biomedical engineering. However, software-based FFT implementations on general-purpose processors suffer from high latency and energy overhead, making them unsuitable for real-time embedded applications. This paper presents the design and implementation of a modular, hardware-accelerated FFT engine tightly integrated with a custom 5-stage pipelined RISC-V processor. The accelerator supports radix-2 Decimation-In-Time (DIT) FFT computation for sizes N = 2, 4, 8, 16, 32, and 64 points using IEEE 754-like half-precision (FP16) floating-point arithmetic. The engine is orchestrated through four custom RISC-V instructions — `FFT.SETN`, `FFT.LOAD`, `FFT.EXEC`, and `FFT.READ` — leveraging the RISC-V custom opcode extension mechanism. Key design features include hardware bit-reversal permutation during data loading, a compact 32-entry twiddle factor ROM, and a time-multiplexed single butterfly unit that minimizes area while supporting arbitrary FFT sizes. A stall-based pipeline integration strategy ensures hazard-free operation without modifying the existing processor hazard unit. The design is verified through cycle-accurate simulation using Icarus Verilog and synthesized using Yosys targeting FPGA platforms. Experimental results demonstrate that the hardware accelerator achieves a [X]× speedup over equivalent software FFT execution while operating within [Y] logic cells and [Z] mW power consumption.

**Keywords** — FFT, RISC-V, hardware accelerator, custom instruction, FP16, half-precision, radix-2, digital signal processing, FPGA
